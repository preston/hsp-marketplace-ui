Quick Start

Let's be brief. If you've even submitted an app to the Apple App Store or Google Play, you're basically looking at the Open Source equivalent with a juicy healthcare twang. The business use cases are very similar to what you're used to. Our focus isn't, however, trying to lock you into a specific vendor ecosystem or cloud provider. The driving values are unapologetic focus on interoperability standards, vendor neutrality, full stack automation, and fair digital rights management.

This system is an implementation of the HL7/Logica Health Services Platform Marketplace 2 specification: a REST API for submission, curation, publication, and automated distrution of computable health products. Ultimately, the aim is to create an networked ecosystem of API-compatible marketplaces, like this one, such that providers and independant developers may freely exchange curated assets without the "walled garden" issues that plague the computing world. And while standards-compliant, plug-and-play health products are great, closed-source assets are just as well. (But they still must meet the same baseline set of packaging and validation requirements as F/OSS products.)

The API is designed for ease of adoption using present-day web and mobile technologies. All resource types are easy to consume using modern webapp frameworks such as Angular and React, and just as easy with Ruby, Python, Swift, Objective-C, Kotlin, Groovy, and anything with decent HTTP/REST support.

Client Developer Overview

This system is not a fluffy academic thought experiment in abstract meta-frameworks for developing conceptual whatevers. It addresses a multitude of provoking HIT ecosystem challenges in a specific way, but isn't a kitchen sink. Grokingly, it is:

  • REST only. No SOAP. HATEOAS principles somewhat apply.
  • Not FHIR endpoints. We support publishing your FHIR app servers, CDS Hooks services, and even raw resource data all day long, but the API itself is not a FHIR thing.
  • JSON only. No XML.. not even HTML. No YML nor CSVs. For all practical purposes it is completely unnecessary to support XML, and it's surprisingly annoying to work with it in complex webapps.
  • Paginated for all indexes, because you have to for high-volume transactional systems.

Open Source infrastructure

The software powering this system is full open source and unencumbered for worldwide adoption and adaptation. The products listed within it are a different story, but the system itself is completely free to use for your own purposes.

Standardized from the ground up.

Pretty much everything you see in this system is an implementation of the HL7 Marketplace 2 specification, jointly developed by Logica Health and derivative of prior works by Dr. Preston Lee.
